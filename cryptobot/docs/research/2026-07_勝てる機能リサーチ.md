# 「絶対に負けない・勝てる機能」徹底リサーチ報告書

作成: 2026-07-12 / 手法: 5角度の並列Webリサーチ(日英) → 3視点(強気クオンツ・懐疑派リスク管理者・コスト実務家)による討論 → 審判による統合。9エージェント・調査ツール74回実行。

## 結論(正直な回答)

**「絶対に負けない・必勝の機能」は作れない。** これは3つの立場すべてが完全一致した結論であり、根拠は三重:

1. **法的・論理的**: 金商法は「必ず儲かる」という断定的判断の提供による勧誘自体を違法とし、金融庁・消費者庁・警察庁、米SEC・CFTCが「必勝AIボット」を詐欺の典型として繰り返し警告(2020年のAI偽装USB業務停止命令、2025年のSECによるAIボットPonzi提訴が実例)。本当に必勝なら他人に売る動機がない、という論理矛盾も本質的。
2. **理論的**: 現物ロングのみである以上BTCの市況リスク(β)は除去不可能。裁定機会も実行コストの裏返しであり資本流入で消滅する(Makarov & Schoar, JFE 2020)。往復コストが存在する限り、期待利幅がコストを下回る売買は数学的に確実な損失。
3. **統計的**: 台湾市場の37億件取引の実証研究(Barber-Odeanら)では、デイトレーダーは手数料込みで平均日次-23.9bpの損失。手数料控除後に継続的に勝てるのは約1%。バックテスト上の「必勝」は多数試行から必然的に生まれる過学習(Bailey & López de Prado)。

**ただし重要な区別**: 「絶対に勝つ」は不可能だが、**「愚かな負け方を構造的に排除する」ことは完全に実装可能**。個人の損失の3分の2以上は執行コストという可変項であり(コスト前-7bp → コスト込み-23.9bp)、手数料負けのゼロ化・破産確率のほぼゼロ化・過学習戦略の実弾投入防止——この3つが10万円・国内現物で確実に取れる唯一の「エッジ」。

**現実的な目標**: 「月利数%」ではなく「1年目: 実効コスト込みでバイ&ホールド±数%を達成し、ドローダウンを圧縮しながら生き残って検証データを貯める」。

## 実装優先度順の機能リスト(討論の合意)

### 1. Post-Onlyメイカー執行エンジン(bitbank post_only + 再指値ループ)
3者全員が最優先で一致した唯一の機能。bitbankのテイカー0.12%→メイカー**-0.02%(手数料を受け取れる)**への転換は往復0.28%の差=市況に依存しない確定的改善。10万円の小口注文は板を動かさないため、メイカー執行では大口より不利になりにくいという小資金唯一の構造的アドバンテージがある。
実装: `post_only:true` で全注文をメイカー強制。N秒無約定で再指値、TTL超過でキャンセル。テイカーへの自動フォールバックはしない(取れなかった利益は仮説、払った手数料は確定損)。手数料率はハードコードせずAPI/設定から取得。

### 2. 多層サーキットブレーカー+手動再開キルスイッチ+執行安全装置
個人botの大損の相当部分は戦略ではなく運用事故(API再送連打・暴走・バグ)。日次損失3-5%/週次7%/連敗5回/最大DD15-20%の4層リミット、発動時は全停止+通知・手動再開必須。注文の冪等化・二重発注防止・価格サニティチェック(±X%乖離で発注拒否)・API断時は「何もしない」フェイルセーフ。
※現行botには単層(日次損失+DD停止)は実装済み。多層化と執行安全装置が追加分。

### 3. 発注前コストゲート+売買頻度ガバナー
期待利幅が往復実効コスト(手数料+スプレッド+スリッページ)のk倍(k≥2)未満の注文を機械的に見送る。加えて月次エントリー回数上限(例: 月4回)、MAクロスは日足以上のみ。短期足シグナルの大半はこのゲートで死ぬはずで、それが手数料負けの事前防止。

### 4. 実効コスト台帳(Implementation Shortfall + バックテスト乖離の自動計測)
注文ごとに理論価格・発注価格・約定価格・手数料・待ち時間を記録し、「ライブPnL−バックテスト想定PnL」の乖離を週次集計。乖離が想定の2倍超で戦略自動停止。全機能の効果測定インフラ。

### 5. ATR連動サイジング+1%ルール+クォーターケリー上限+最小注文単位チェック
size = (資金×1%) ÷ (ATR14×2〜3)。ケリー推定値の25%を上限、ケリー値マイナスなら新規停止。算出サイズが最小注文単位未満なら「切り上げ」ではなく「見送り」(切り上げは1%ルールの静かな崩壊)。

### 6. 過学習検出デプロイゲート(ウォークフォワード+Deflated Sharpe+段階昇格制度)
全パラメータ試行回数を記録しDSR・PBOを計算。「ペーパー→最小ロット実弾→通常運用」の段階昇格、乖離超過で自動降格。最大の「リスク」はMAクロスがゲートを通らないと判明することだが、それは機能であって欠陥。

### 7. 税務対応記帳の拡張(総平均法対応+税引後ベンチマーク比較)
個人の法定デフォルトは**総平均法**(移動平均法は届出制・3年変更不可)のため両対応が必要(現行は移動平均法のみ)。月次で「戦略の税引後 vs DCA+HODL vs B&H」を自動比較。税引前+5%の高回転戦略は税引後でDCAに負けうる。

### 8. 200日MAレジームフィルター+DCA傾斜(スマートDCAコア)
唯一の「攻め」枠だが、リターン向上ではなくドローダウン圧縮の機能。価格が200日MA未満では新規買い停止・DCA減額、以上でのみMAクロス有効。DCAは200日MAからの乖離率で購入額を±50%傾斜(安い時に多く買うの機械化)。Liu & Tsyvinski(NBER)のモメンタム実証等、現物ロングオンリーで再現可能な数少ない学術的裏付けのある系統。

## 警告(騙されやすい罠・やってはいけないこと)

1. **「必勝」「AI搭載」「元本保証」「月利◯%保証」を謳う商材・勧誘は、その宣伝文句自体が金商法違反のシグナル**。実例: AI非搭載54万円USBのマルチ商法(業務停止命令)、出金時に追加請求するPonzi型。困ったら消費者ホットライン188、金融庁 0570-050588。
2. **販売所を絶対に使わない**(スプレッド実測4-6%、1回で年間期待リターンが飛ぶ)。必ず取引所(板取引)+指値。
3. **バックテストの好成績を信じない**。バックテスト250%→実運用-30%の報告例あり。検証ゲート通過まで「まだ存在しないリターン」として扱う。
4. **成績が出ない時の「高頻度化・銘柄追加・レバレッジ・海外取引所」への誘惑に乗らない**。高頻度化は手数料・滑り・課税イベントの三重コスト増。
5. **税金を後回しにしない**。雑所得(最大約55%)・損失繰越不可。申告時にまとめて計算して税引後赤字に気づくのでは1年遅い。

## 討論で対立した論点と決着

- **期待値マイナスは宿命か可変か** → 同じ台湾研究の表裏。コスト反転は確定的改善だがアルファではない。目標は「1年目はB&H±数%+DD圧縮+生存」で3者合意。
- **テイカー・フォールバックの是非** → 懐疑派の「原則なし」を採用。テレメトリで逆選択コストを実測してから見直す条件付き。
- **MAクロスは戦略かフィルターか** → 「リターンを稼ぐ戦略」から「200日MAレジームフィルター=DD圧縮装置」に再定義。
- **検証の徹底 vs 学習速度** → 段階昇格制度で両立。実弾は「学習」ではなく「乖離測定という仮説検定」のために最小額で使う。
- **税引前 vs 税引後・計算方法の矛盾** → 記帳は総平均法・移動平均法の両対応、戦略評価は税引後で。
