## 4. 数字のリアル（誇張しない）

- **現金コスト：ほぼゼロ**。らくスレ・Threads・LINEは無料枠で回る。有料ドメイン/私書箱/ツールは「有料商品を売り始める時」まで不要。→ **年 数千〜数万円**。
- **収益：小さい**。529フォロワー・月0〜数万の楽天アフィが現実線。有料商品が当たっても月数万レンジ。
- **最大のコストは"時間"**。だから判断基準はシンプル：
  - **余暇（夜・週末）でやる → 機会費用≒¥0 → やる価値あり（現金リスクほぼゼロ・小さく黒字・複利で伸びうる）。**
  - **本業（漫画/IP）の時間を削る → NO。** 本業のほうが時間対効果が高い。
- **結論：余暇の範囲で続ける限りGO。** 大きく張らない・焦らない。

## 5. 90日でやること（ゼロ円・余暇）

「新しく作る」より「**今あるものを回す**」。週5〜7時間まで。

1. **らくスレを使ってもらう**：@tsukuru.laboratory の固定ポスト／プロフィールから LINE → らくスレ配布（既にある導線の文言を磨く）。
2. **@tsukuru.laboratory を育てる**：4本柱（①発見共感 ②Build in Public ③実演 ④正直・逆張り）で1日1〜3投稿。**リンクは5回に1回・#PR必須**。→ `../04-execution/threads-growth-playbook.md`
3. **LINEを整える**：登録直後〜数通の"価値提供"を用意（押し売りしない）。→ `../04-execution/line-funnel-monetization.md`
4. **数える**：LINE登録数・らくスレを使った人・保存/返信。フォロワー数は追わない。
5. **有料はまだ作らない**：需要（LINEの反応）が見えてから、1つだけ・10本テスト。

## 6. リスクと守ること（"設計"で回避）

| リスク | 対処 |
|---|---|
| **楽天/Threads の BAN・規約厳格化** | らくスレは**手動コピペ・自動投稿なし**。毎投稿リンクを貼らない（3〜5:1）。**LINEリスト**をBAN耐性の避難先にする。 |
| **無料代替が強い**（Rinker等） | 「Threads特化＋"めんどくさいを消す"体験＋#PR自動」で差別化。勝てなければ無理に有料化しない。 |
| **ステマ規制（景表法）** | 全投稿・LINEで **#PR／広告表記**。らくスレは #PR デフォルト。 |
| **特商法・商標** | 有料販売の時だけ特商法表記（法人名・連絡先・返金）。**rakusure.com に「楽天と無関係／下書きなので投稿前に規約確認」の免責を掲載**（オーナー対応）。 |
| **誇大・情報商材化** | 「1ヶ月で月◯万」「必ず稼げる」を言わない。"めんどくさいを消す・正直"の立ち位置を守る（これ自体が信頼＝差別化）。 |
| **本業を侵食** | 同時稼働≤2・余暇キャップ。超えたら止める。 |

## 7. やらないこと（レッドライン）

企業への営業／実名リストの無同意販売／特定個人を名指しで「誇大」と断定した有料化／「◯万達成」等の即金・再現性の訴求／自動投稿・スクレイピング／**お金をかける（当面）**。
