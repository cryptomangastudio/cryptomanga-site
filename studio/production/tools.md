## 工程ごとの推奨

### B/C デザイン・生成・一貫性ロック
かわいい系マスコット（単純形状・フラットカラー）は一貫性を出しやすい。**まず学習不要の参照ベースで十分**、量産段階でLoRAへ。

| ティア | ツール | 使いどころ |
|---|---|---|
| 参照ベース（学習不要・即production） | **Ideogram Character** / **Midjourney（character reference/omni-reference）** / Mage(Characters) | 検証フェーズの標準。1枚の基準画から同一キャラを量産 |
| 学習ベース（最高の同一性・要手間） | **LoRA**（ComfyUI / Kohya_ss、Stable Diffusion / FLUX系） | ロック済み15〜50枚で学習（1000〜3000 step, 1〜4h）。数百枚規模の量産・スケール後 |
| プロの合わせ技（スケール/アニメ） | 低強度LoRA + PuLID + ControlNet(OpenPose) | 全フレーム同一が要る動画/大量スプライト。検証段階ではオーバースペック |

- 2026の潮流：FLUX.2 / GPT Image系の登場で、**学習ベースと参照ベースの差は縮小**。多くの用途は参照ベースで足りる。
- 選定の実務：候補2ツールで**同じキャラを10枚**出し、同一性・かわいさ・破綻率で比較してから固定。

### F ベクター化・商品化
| 用途 | ツール |
|---|---|
| raster→SVG / text→vector | **Recraft（V4.1）** … 任意サイズで破綻しない真のSVG。フラットカラーのキャラ・**ステッカー/グッズ/大判印刷**に最適 |
| 手仕上げ | Illustrator / Figma（SVGを再編集） |

## 選定チェックリスト（新ツール採用時）
