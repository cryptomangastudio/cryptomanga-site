# 制作パイプライン（キャラを"どう作るか"）

原則：**ツールは半年で変わる。だからツール非依存の"工程"を固定し、ツールは差し替え可能にする。**
資産は必ず `characters/<name>/assets/{references,raw,clean,vector}` に流し込む。

```
[A 企画/設計]→[B マスターデザイン]→[C 一貫性ロック]→[D 量産作画]→[E 清書]→[F ベクター化/商品化]
   人間          AI＋人間ディレクション      AI              AI＋人間      人間/AI      AI(Recraft)
```

## A. 企画・設計（人間＝ここが魂。AIにやらせない）
- `character-bible.md` を埋める：性格・生い立ち・世界観・**共感フック**（すみっコ型の"ちょっとネガティブ"）。
- creative-writing スキルを使う。**キャラの魂＝差別化＝堀**。ここが弱いと以降すべて無駄。

## B. マスターデザイン（1体につき1回）
- キャラの"正解の姿"を確定させる基準画を作る。
- 出力先：`assets/references/master.png`
- ツール：`tools.md` の「デザイン/生成」ティア参照。かわいい系マスコットは**単純な形＝一貫性を出しやすい**。

## C. 一貫性ロック（キャラシート）
- マスターから **ターンアラウンド（前/横/後）＋表情差分＋基本ポーズ** を作り、"канон(正典)"を固定。
- 出力先：`assets/references/`（`turnaround.png` `expressions.png` `poses.png`）
- 方式（`tools.md`参照）：
  - **検証フェーズ＝参照ベース**（キャラリファレンス機能）。学習不要で即production-ready。
  - **量産/スケール後＝LoRA学習**（ロック済み15〜50枚で学習）。高ボリュームで同一性が要る時。

## D. 量産作画（4コマ・SNS・ポーズ）
- Cのキャラシートを参照に量産。**コマ割り・構図・セリフ＝人間が演出**（＝マンガの本業）。
- 出力先：`assets/raw/`、4コマは `content/4koma/`

## E. 清書
- 破綻（手・線・アーティファクト）を人間が修正。出力先：`assets/clean/`

## F. ベクター化 / 商品化
- グッズ・LINEスタンプ・大判印刷は**ベクター(SVG)必須**。Recraftで raster→SVG or text→vector。
- 出力先：`assets/vector/`。Illustrator/Figmaで再編集可能な状態を保つ。

## 資産管理の鉄則（キャラIP＝資産管理業）
- **再現できるように記録する**：使ったモデル・seed・プロンプト・設定を必ず `prompt-library.md` に残す。
- 1キャラ＝1フォルダ。命名は連番＋内容（例：`4koma_007_shoulder.png`）。
- `references/` は"正典"。ここを汚さない（生成の当たり外れは `raw/` に置く）。
