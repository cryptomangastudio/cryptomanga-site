# Kindle × note ビルダー

CryptoManga Studios のKindleとnoteのB2C出版事業を、**創業者の手を最小化しながら、規約に通り・読了され・売れる**品質で回すための実務スキル。徹底リサーチ(複数回・敵対的検証つき)で確定した事実に基づく。

## 大前提(最初に思い出すこと)

1. **量産で薄く稼ぐは成立しない**。KENP単価≒0.478円/ページ、KDPは新規1日3点まで、手軽なジャンルは凍結多発。→ 冊数でなく「**当たる型を1つ作って複製**」。
2. **握るのは①企画の判断 と ④編集の監修 だけ**。②③⑤⑥⑦はAI・テンプレ・外注で回す。「手を減らす=創作の消耗を減らす」であって「品質チェックを飛ばす」ではない。
3. **AI利用は正直に開示**(開示は読者に見えない=ノーリスク)。ただし人間の創作的関与を厚くして"AI補助"の水準に持っていくと、開示義務の外+品質+著作物性+規約遵守を同時に満たす。
4. **有力な事業の軸 = 逆張り「副業やめとけ」×実体験の人格課金**。月10万が現実目標、月50万は非保証。

## 標準の進め方

ユーザーが「Kindle本/note記事を作りたい」と言ったら:

1. **企画を固める**(①)。`kindle/templates/書籍企画書.md` を埋め、`02`のGo/No-Go判定。逆張り軸なら`06`のタイトル型・出口テーマ接続を使う。
2. **お作法を確認**。`references/お作法.md` で、そのフォーマット(実用書/エッセイ/小説/note有料記事)の推奨文字数・体裁・冒頭の型・価格/KUの初期値を取る。
3. **骨子→執筆→編集**(②③④)。`kindle/workflow/プロンプト集.md`のプロンプトを章単位で回し、【一次情報】【要確認】マーカーを④で潰す。**編集の監修は必ず人間(ユーザー)に返す**。
4. **表紙・販売ページ**(⑤)。`templates/表紙ブリーフ.md`『販売ページ.md』。
5. **入稿前チェック**(⑥)。`kindle/docs/出版チェックリスト.md`を上から。AI利用は`templates/AI利用記録.md`に記録し正直に開示。
6. **記録**(⑦)。`kindle/tracker/書籍マスター.csv`に登録、`収益ダッシュボード.md`を更新。

## やってはいけないこと

- AI出力をそのまま入稿する(事実確認・独自性注入を飛ばす)
- 「必ず/絶対/誰でも100%」等の誇大・断定
- タイトル/表紙と中身の不一致(釣り)
- レビューの自作自演・購入・交換(規約違反)
- 手軽に見えるジャンル(AI写真集・アダルト隣接)への安易な参入(凍結=全滅)
- 煽り系の収益成功譚(note月40万等)を鵜呑みにして計画に織り込む(検証で全て棄却済み)

## 数字は変わる

KENP単価・KDP規約・note手数料・税制は変動する。金額が大きい判断の前に、規約は公式(KDPヘルプ)、税務は税理士で最新を確認する。このスキルは事実の"最終確認先"ではなく"設計の土台"。
