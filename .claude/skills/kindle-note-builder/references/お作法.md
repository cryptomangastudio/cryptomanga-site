# Kindle・note お作法リファレンス(制作実務の初期値集)

制作者が毎回参照する「迷ったときの推奨デフォルト+なぜ」の一覧。

> **このリファレンスの確度について(正直に)**: Kindle/noteの制作お作法を徹底リサーチしたが、権威ある一次ソース(KDP公式ヘルプ kdp.amazon.co.jp、note公式ヘルプ、実務ブログ shuppan-lab 等)は**作業環境の組織egressポリシーで本文取得が遮断**され、自動検証を通せなかった。そのため本書は**確立された実務知識に基づく推奨初期値**であり、★印の**精密仕様(表紙のピクセル・手数料率・価格上限・キャンペーン日数など)は下記「確認先」で実行前に必ず最新を確認**すること。値は目安・慣行であり、規約は変わる。
>
> **確認先(実行前に開く)**:
> - KDP 表紙/原稿/仕様: kdp.amazon.co.jp のヘルプ(トピック G200645690 / G201605710 / G200645680 / G200634390)

## 1. 文字数・分量の設計

| フォーマット | 推奨初期値 | なぜ |
|---|---|---|
| 実用書(単品売り) | **15,000〜30,000字** | 薄すぎ(1万字未満)は「低コンテンツ」疑い+返品リスク。厚すぎは冗長で読了率低下。悩み1つを解ける密度が最適 |
| 実用書(KU/KENP狙い) | **20,000〜40,000字(厚め)** | KENPは「実際に読まれたページ×約0.478円」。分量は収益の上限を上げるが、**水増しは読了率を下げて逆効果**。密度を保ったまま厚く |
| エッセイ | 10,000〜30,000字 | 読み物としてのまとまり。短編連作も可 |
| 小説 | 40,000字以上が一般的 | 短編集・連載形式もあり。ライト価格帯で数万字 |
| note有料記事 | **3,000〜10,000字** | 「無料で読める情報以上の密度」が課金の条件。短くても一次情報・具体・再現性があれば売れる |

**設計の原則**: 単品売りは「1回の購入」で収益確定 → 分量より **買う理由(タイトル/表紙/説明/冒頭サンプル)** が効く。KU/KENPは **最後まで読ませる牽引力 × 相応の分量** の両立が収益。

## 2. 書き方の作法(スマホ縦画面が前提)

- **章立て**: 見出しは **章(見出し1)/ 節(見出し2)** の2階層を基本。見出しスタイルから目次を生成する(3.参照)。
- **1文は短く(目安40〜60字)、1段落2〜4文**。スマホの縦画面で「文字の壁」を作らない。
- **余白・小見出し・箇条書き・太字**で要点を可視化。段落間に適度な改行。
- **冒頭(試し読みサンプル=通常本文の約10%)で離脱させない ★**:
  - 最初の数行で「**誰の何が解決するか(ベネフィット先出し)**」。
  - 長い自分語り・謝辞・前置きから始めない。まえがきは短く「価値の約束」に絞る。
  - 実用書は「この本を読むと、読む前( )→ 読んだ後( )」を早々に提示。
- **note売れる構成の型**: フック(共感/逆張り)→ 権威(実体験・実績)→ 無料で価値の一部 → **有料で核心(手順・具体・データ)** → 読後の変化。

## 3. 入稿の技術仕様 ★(数値は確認先で最終確認)

- **原稿フォーマット**: Word(.doc/.docx)または EPUB。Kindle Create で Word を整形するのが手軽。実用書/小説は **リフロー型**、写真集/漫画は固定レイアウト。
- **目次(TOC)**: 本文の **見出しスタイル(見出し1/2)で構造化** → 論理目次(NCX/HTML TOC)が生成される。「紙のページ番号」でなく **見出しジャンプ**にする。冒頭に目次ページも置きリンクを張る。
- **表紙 ★**: 縦横比 **1.6:1(高さ:幅)** 推奨。理想サイズの目安 **2,560×1,600px**、長辺は最低1,000px以上。JPG/TIFF。**サムネイル(小)で読める大きな文字**が最優先。
- **画像/挿絵**: 解像度と総ファイルサイズに注意。**70%ロイヤリティでは配信コスト(ファイルサイズ課金)が差し引かれる**ため、無駄に重い画像を避ける。

## 4. 発見される技術(検索・回遊・転換)

| 要素 | 推奨初期値 | なぜ |
|---|---|---|
| タイトル | 主要検索語を**自然に**含める。誇大・記号乱用NG | 検索一致+クリック。詰め込みは規約・可読性ともにNG |
| サブタイトル | ベネフィット+読者+検索語 | タイトルで拾えない語を補完 |
| カテゴリ ★ | 出版時に選択+**KDPに追加申請でニッチカテゴリに登録** | 競合の薄いニッチで「ベストセラー1位(オレンジタグ)」を狙える |
| キーワード | **7枠**を使い切る。タイトル未収録の検索語・読者の言葉・状況語 | 発見面を広げる。「無料」「ベストセラー」等の**禁止語・商標は避ける** |
| 価格 ★ | **¥250〜¥1,250 で70%ロイヤリティ**(範囲外は35%)。初値¥250〜¥500 | 価格帯を外すと印税が半減。低め初値で最初の実績を作る |
| KDP Select ★ | KENPで稼ぐなら**加入(90日単位・他ストア独占)** | KU/KOLL対象+無料キャンペーン/Countdown Dealが使える |
| シリーズ(Series) | 続き物は Series でまとめる | 回遊・次巻購入を促進(量産の勝ち筋=シリーズ) |
| A+コンテンツ | 商品ページに画像リッチな説明を追加 | 転換率向上 |

## 5. 発売直後の初速 ★

- **キャンペーン**: KDP Select加入中は **90日ごとに「無料キャンペーン(数日)」か「Countdown Deal」のどちらか**。無料DLでランキング露出→有料に戻して初速を作る(日数・回数は確認先で最新確認)。
- **告知の集中**: 発売前にSNS/noteで予告し、**初日にアクセスを集中**。ランキングは初速を評価する。
- **レビュー獲得の作法(やってよい)**: 読者へ正直に「感想をいただけると嬉しい」と依頼する。
- **やってはいけないレビュー操作(規約違反=削除+アカウントリスク)**: 対価・商品提供と引き換えのレビュー、レビューの購入、自作自演、レビュー交換、関係を開示しない身内レビュー。

## 6. note の作法 ★

- **有料記事の文字数**: 3,000〜10,000字が目安。密度が命(1.参照)。
- **無料/有料の境界**: 冒頭を無料で読ませ「価値の予告+結論の一部」を提示、**核心(手順・具体・データ)を有料に**。境界直前で「この先に何があるか」を明示。
- **見出し画像 ★**: **1,280×670px** 推奨。内容が伝わり目を引くもの。
- **価格 ★**: 単発 **¥100〜**(本人確認済みで上限が上がる)。初値 ¥300〜¥1,000 が試されやすい。
- **マガジン/メンバーシップ**: 複数記事をまとめる「マガジン」、月額の「継続購読マガジン/メンバーシップ」で**ストック収入**を作る(単発より安定。ただしファン基盤が前提)。
- **手数料 ★(要確認・未確定)**: プラットフォーム手数料+決済手数料が売上から引かれる。**実率は変動するため note公式ヘルプで必ず確認**(本フォルダの研究でも確定できなかった項目)。

## 7. AI活用時の編集チェック(規約通過+品質担保)

出版・公開前に必ず通す。`kindle/docs/出版チェックリスト.md`と対で使う:

- [ ] **事実確認**: AI由来の固有名詞・数値・日付・法律・手順を1つずつ検証(ハルシネーション除去)
- [ ] **重複・冗長の圧縮**: AI特有の言い換え反復を削る(読了率・KENPに直結)
- [ ] **独自性の注入**: 一次情報(実体験・独自例・データ)を最低1〜3箇所 → **"AI補助"の水準に上げ、開示義務の外+品質+著作物性を同時に満たす**
- [ ] **トーン統一**: シリーズは語り手を固定
- [ ] **引用・権利・免責**: 引用ルール、商標、画像ライセンス、医療/投資/法律の断定回避+免責文
- [ ] **AI開示の判定**: 「AI生成(要開示)」か「AI補助(不要)」かを`templates/AI利用記録.md`で判断し、迷ったら開示側に倒す
- [ ] **低品質判定の回避**: 釣りタイトル・検索記事の焼き直し・薄さ・内容とメタデータの不一致がないか
