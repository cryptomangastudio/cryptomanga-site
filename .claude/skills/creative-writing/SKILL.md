# 創作(小説・マンガシナリオ)スキル

このスキルは、物語を「思いつきで書く」のではなく、**編集者としての設計**と**書き手としての筆力**の両方を持って創作するためのもの。CryptoManga Studiosの実務(シナリオ制作・編集・制作進行)を念頭に置いている。

## 使い方の原則

1. **いきなり本文を書き始めない。** 依頼の粒度に応じて「プロット→構成→本文」の順で段階を踏む。相手が明らかに本文だけを求めている短い依頼(掛け合いセリフ1本など)は除く。
2. **関連するreferenceファイルを必要な分だけ読む。** 全部を毎回読み込む必要はない。下の「参照ガイド」から該当するものを選んで読む。
3. **書いたら自分で一度疑う。** 書きっぱなしにせず、`references/editing-checklist.md` の観点で最低1周は自己チェックする。
4. **媒体を確認する。** 小説(縦書き想定の地の文中心)なのか、マンガ原作(絵で見せる前提)なのか、WEBTOON(縦スクロール)なのかで、書き方の作法がまったく違う。不明なら聞くか、文脈から推定して明示する。

## 参照ガイド(必要に応じて読む)

- **`references/story-structure.md`** — プロット・構成の作り方。起承転結、三幕構成、ヒーローズジャーニー、連載の話数設計、引きの作り方。「話の骨組みを考えて」「プロットが薄い」というときに読む。
- **`references/character-craft.md`** — キャラクター設計。欲求と欠落、関係性設計、キャラクターボイス、成長曲線。「キャラが立ってない」「主人公を作って」というときに読む。
- **`references/manga-scenario-format.md`** — マンガ・WEBTOON原作の実務フォーマット。プロット→ネーム→台本の書式、コマ割り指示の書き方、見開き・縦スクロールの演出設計。**マンガ/WEBTOONの仕事はここが最重要**。
- **`references/prose-and-dialogue.md`** — 小説の地の文・セリフの文章技術。視点(POV)、Show don't Tell、セリフのサブテキスト、リズム。
- **`references/genre-playbook.md`** — ジャンル別のお約束と外し方(少年・少女・青年・異世界・恋愛・サスペンス等)。「テンプレっぽくなる」「王道だけど新しさが欲しい」というときに読む。
- **`references/editing-checklist.md`** — 推敲・編集チェックリスト。原稿を書き終えた後、必ずこの観点で見直す。

## 進め方の型

依頼が「新しい物語を作って」に近いほど、以下の順で段階的に進める(全部を毎回律儀にやる必要はなく、依頼の規模に応じて圧縮してよい):

1. **ログライン**(1〜2文で「誰が」「何を望み」「何が邪魔するか」)
2. **プロット**(`story-structure.md` 参照。全体の起伏)
3. **キャラクター**(`character-craft.md` 参照。主要人物の欲求・欠落・関係)
4. **構成/話数割り**(連載なら各話の引き。単発なら三幕の配分)
5. **本文 or ネーム**(媒体に応じて `manga-scenario-format.md` か `prose-and-dialogue.md`)
6. **自己レビュー**(`editing-checklist.md`)

段階ごとに一度立ち止まり、ユーザーに方向性を確認してよい箇所(トーン、対象年齢、既存作との差別化など)は先回りして提案しつつ、大きな分岐は確認する。
