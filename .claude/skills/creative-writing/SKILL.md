# 創作(小説・マンガシナリオ)スキル

このスキルは、物語を「思いつきで書く」のではなく、**編集者としての設計**と**書き手としての筆力**の両方を持って創作するためのもの。CryptoManga Studiosの実務(シナリオ制作・編集・制作進行)を念頭に置いている。

## 使い方の原則

1. **いきなり本文を書き始めない。** 依頼の粒度に応じて「プロット→構成→本文」の順で段階を踏む。相手が明らかに本文だけを求めている短い依頼(掛け合いセリフ1本など)は除く。
2. **関連するreferenceファイルを必要な分だけ読む。** 全部を毎回読み込む必要はない。下の表から該当するものを選んで読む。
3. **書いたら自分で一度疑う。** 書きっぱなしにせず、`references/editing-checklist.md` の観点で最低1周は自己チェックする(推敲は構造→キャラ→媒体→文章の順)。仕上げに `references/anti-ai-tells.md` で「整いすぎ」を崩す——構造的に正しいことと、人間の筆跡に見えることは別問題。
4. **媒体を確認する。** 小説(地の文中心)/マンガ原作(絵で見せる前提)/WEBTOON(縦スクロール)で作法がまったく違う。不明なら聞くか、文脈から推定して仮定を明示する。
5. **長い納品物はいきなり全部書かない。** プロット・方向性の段階で一度ユーザーに見せて確認を挟む(手戻り防止。`pitch-planning.md` の粒度管理参照)。

## 依頼タイプ → 参照ガイド

| 依頼の種類 | 読むファイル |
|---|---|
| 話の骨組み・プロット・話数構成・引きが弱い | `references/story-structure.md` |
| キャラを作る・キャラが立たない・敵役設計 | `references/character-craft.md` |
| 世界観・設定・魔法/能力のルール・組織や勢力 | `references/worldbuilding.md` |
| マンガ/WEBTOONの台本・ネーム・コマ割り指示 | `references/manga-scenario-format.md`(**マンガ案件の最重要ファイル**) |
| 小説本文・地の文・セリフの執筆と改稿 | `references/prose-and-dialogue.md` |
| ジャンルのお約束・テンプレ回避・差別化 | `references/genre-playbook.md` |
| 企画書・あらすじ・持ち込み資料・編集フィードバック | `references/pitch-planning.md` |
| 書き上げた後の推敲・レビュー依頼 | `references/editing-checklist.md` |
| 「AIっぽい」「整いすぎ」と言われた・納品前の最終パス | `references/anti-ai-tells.md`(**すべての納品物に最後に必ずかける**) |

複合的な依頼(「新連載の企画を丸ごと」等)は複数ファイルを組み合わせる。典型的な組み合わせ:

- **新規マンガ企画**:pitch-planning + story-structure + character-craft + genre-playbook
- **第1話の台本執筆**:manga-scenario-format + story-structure(掴みと引き)
- **小説の新作**:story-structure + character-craft + prose-and-dialogue
- **原稿への編集レビュー**:editing-checklist + pitch-planning(フィードバックの書き方)

## 進め方の型

依頼が「新しい物語を作って」に近いほど、以下の順で段階的に進める(依頼の規模に応じて圧縮してよい):

1. **ログライン**(1〜2文。「誰が」「何を望み」「何が邪魔するか」+できれば皮肉)
2. **プロット**(全体の起伏。ミッドポイントとオールイズロストを先に決める)
3. **キャラクター**(主要人物のWant/Need/Wound と関係性)
4. **世界観**(必要なら。物語を駆動する「制約」だけ固める)
5. **構成/話数割り**(連載なら各話の引き。単発なら幕の配分)
6. **本文 or 台本**(媒体に応じた作法で)
7. **自己レビュー**(editing-checklist で構造→キャラ→媒体→文章の順)

段階ごとに、方向性の分岐(トーン、対象読者、既存作との差別化など)は先回りして提案しつつ、大きな分岐はユーザーに確認する。

## 品質の合言葉

- **予想は裏切り、期待は裏切らない**(ジャンルが約束する感情は必ず届ける)
- **すべてのシーンは何かを変える**(状況・関係・認識のどれか)
- **絵で語れることをセリフで潰さない**(マンガ/WEBTOON)
- **設定は制約から、開示は読者が知りたくなった瞬間に**
- **フィードバックは道筋を渡す**(批評で終わらせない)
