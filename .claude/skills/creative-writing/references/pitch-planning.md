# 企画書・あらすじ・持ち込み資料の作り方

スタジオ実務では「面白い話を作る」のと同じくらい「**企画を通す・伝える**」文書力が問われる。編集部・クライアント・作画者に向けた資料の作法をまとめる。

## 1. 企画書の基本構成(1〜2枚に収める)

```
1. タイトル(仮)+キャッチコピー1行
2. ログライン(1〜2文。皮肉/フックを含む)
3. ジャンル・媒体・対象読者(誌面/WEBTOON/アプリ、年齢層・性別傾向)
4. 企画意図(なぜ今これか。読者のどんな欲求に応えるか)
5. 類似作と差別化(代表作を2〜3挙げ、「◯◯は共通、△△が違う」を一言で)
6. 主要キャラクター(主人公+2〜3人。Want/Needと関係の一言ずつ)
7. 全体の流れ(序盤・中盤・終盤を各2〜3行。連載なら想定話数・アーク構成)
8. 第1話のあらすじ(掴みと引きが分かる粒度で)
```

- **1枚目の上半分で興味を持たせられなければ読まれない**。ログラインとキャッチに最も時間をかける。
- 「差別化」欄は謙遜も誇張もせず具体で書く。「今までにない斬新な物語」のような自己評価の形容詞は信頼を下げる。
- 絵の企画(マンガ)は、可能なら「絵になる場面」を言葉で3つ挙げる(ビジュアルの想像がつく企画は通りやすい)。

## 2. あらすじの書き分け(用途で全く別物)

- **企画用あらすじ(〜400字)**:結末まで全部書く。「続きはお楽しみ」は禁止——選考者・編集は結末の設計力を見ている。時系列順に、主人公の選択と変化が分かるように。
- **宣伝用あらすじ(100〜200字)**:結末は伏せる。フック(疑問・違和感・魅力的な状況)で終える。ターゲット読者が「自分向けだ」と分かる言葉(ジャンル記号)を入れる。
- **梗概(シノプシス、800〜1200字)**:公募・原作提供向け。全プロットを幕ごとに要約。ここでも結末まで書く。

## 3. ネーム・原稿への編集フィードバックの書き方

受け手(作家)が動ける形で返す。批評ではなく改稿の道筋を渡す。

- **順番**:①良かった点を具体で(社交辞令でなく、残すべき箇所の特定として)→②最重要の直し1〜3点→③細部リスト。
- 直しは「なぜ機能していないか(読者にどう見えるか)」+「改善の方向性(案は複数あってよい)」のセットで。**答えの押し付けではなく選択肢の提示**が基本。ただし明確な技術ミス(視点ブレ、作画不能な指定、矛盾)は端的に指摘してよい。
- 作家の意図が読めない箇所は、直しを出す前に「ここはどういう狙いですか」と確認する。意図を潰す直しは信頼を失う。
- 全部に赤を入れない。1回の改稿で直せる量(最重要3点まで)に絞り、残りは次のラウンドへ。

## 4. 制作進行のための粒度管理

- 納品物の定義を最初に合意する:プロットのみ/箱書き/台本/ネーム構成案のどこまでか、文字数・ページ数・話数、締切。
- 中間確認のポイントを設ける:「プロットOK後に台本着手」のように、手戻りが大きい工程の前に確認を挟む設計にする。ユーザーとのやりとりでも同じで、長い納品物をいきなり全部書かず、プロット段階で方向確認を入れる。
- 改稿は**バージョンと変更点を明示**する(「v2:編集指摘に対応し、2話の引きを反転型に変更」)。何をなぜ変えたかの履歴が積もると、議論が後戻りしない。

## 5. 権利・オリジナリティ上の注意

- 既存作品の設定・キャラクター・固有名詞をそのまま流用しない。「◯◯風」の依頼は、その作品の構造・ジャンル文法を参照するに留め、表現(セリフ・固有の設定・展開の具体的な組み合わせ)は独自に作る。
- 実在の人物・団体・事件をモデルにする場合は、その旨と権利・名誉毀損リスクの確認事項をユーザーに明示する。
- 職業・医療・法律などの専門描写は「フィクションとしての設計」と「事実確認が必要な箇所」を区別して納品時に注記する。
