# キャラクター設計

## 1. 欲求(Want)と欠落(Need/Wound)を分けて設計する

- **Want(欲求)**:キャラクターが自覚して追いかけているもの。プロットを駆動する(優勝したい、あの人に告白したい、復讐したい)。
- **Need(欠落/内的な傷)**:本人が自覚していない、本当に必要としているもの。多くの場合Wantと衝突する(復讐を果たすことより、誰かを許すことが本当は必要、など)。
- 物語のクライマックスは「Wantを諦めてNeedを取る」か「Wantを掴んだ結果Needも満たされる」かの選択になることが多い。この対立構造がないと、キャラクターが「目的を叶えて終わり」の平坦な話になる。

主要キャラクターを作るときは必ずこの2つをセットで言語化する。脇役でも主要な仲間・敵には最低限用意する。

## 2. キャラクターは「関係性の中の役割」で立たせる

単体の設定(強い/かわいい/クール)ではキャラは立たない。**他のキャラとの関係でどう振る舞うか**で初めて立つ。

- 主人公と対になる**フォイル(対照キャラ)**を最低1人用意する。似た境遇で違う選択をした人物、価値観が正反対の人物など。
- 全員が同じ話し方・同じ価値観にならないよう、グループ内で「誰が何を代弁する係か」を割り振る(理性の係、本音の係、規範の係、混沌の係、など)。
- 恋愛・友情・師弟などの関係性は、**力関係が固定されない**ように動かす。常に片方が助ける側/助けられる側だと関係が停滞する。

## 3. キャラクターボイス(セリフの個性)

- 語尾や口癖だけで差別化しない(表層的ですぐ飽きられる)。**価値観・話す速度・省略の仕方・比喩の選び方**で差をつける。
- 「このセリフ、名前を隠しても誰が言ったか分かるか?」を自己チェックの基準にする。
- 沈黙・言い淀み・話題を逸らす、といった「言わないこと」もキャラクターボイスの一部。

## 4. 成長曲線(キャラクターアーク)

- **ポジティブアーク**:欠落を克服し、より良い自分になる(王道の主人公アーク)。
- **フラットアーク**:キャラクター自身は変わらないが、その価値観が周囲や世界を変える(信念を貫く系の主人公向き)。
- **ネガティブアーク**:欠落に負けて堕ちていく(敵役や悲劇の主人公向き)。
- 主要キャラには最初にどのアークを取るか決めておく。連載途中で変える場合も、読者に「なぜ変わったか」の因果を提示する。

## 5. 敵役・障害となる人物の設計

- 敵は「主人公の価値観を最も強く否定する存在」であるほど物語が締まる。単なる強さの比較にしない。
- 敵にも独自のWant/Needを与える。敵から見た物語でも筋が通るくらいの動機があると深みが出る(全面的な共感は不要。理解できれば十分)。
- 「主人公と敵が同じ問題に対して違う答えを出した」構造(ダークミラー)は、テーマを際立たせる定番の型。

## 6. キャラクター設計シートの雛形(依頼に応じて使う)

- 名前・年齢・立場
- Want(表の目的)
- Need(内的な欠落・本当に必要なもの)
- 欠落の原因(過去のどんな出来事から来ているか)
- 一人称・口調・話し方の特徴(価値観がにじむ選び方をする)
- 得意なこと/致命的に苦手なこと
- 他の主要キャラとの関係と、その関係がどう変化していくか
- このキャラが物語の中で担うテーマ的な役割
