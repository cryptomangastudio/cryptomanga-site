# 推敲・編集チェックリスト

原稿(プロット・台本・小説本文いずれも)を書き終えたら、最低1周はこの観点で見直す。全項目を毎回律儀にやる必要はないが、依頼の重要度に応じて厚みを調整する。

**推敲は上の層から順に**。文章を磨いてから構造を直すと、磨いた文章ごと捨てることになる。マクロ→キャラ→媒体→文章の順で見る。全チェックを通した後、最後に `anti-ai-tells.md` で「整いすぎ」を崩す仕上げパスをかける。

## マクロ(構造レベル)

- [ ] ログラインを1〜2文で言い直せるか。物語の芯がぶれていないか
- [ ] 主人公は受け身になっていないか(事件に反応するだけでなく、能動的に選択しているか)
- [ ] 「転」または中盤の転換で、価値観・関係性が実際に変化しているか
- [ ] シーンを1つ抜いても話が成立してしまう箇所はないか(=何も変えていないシーン)
- [ ] 各話・各章の終わりに引きがあるか。引きの型(危機/謎/決意/反転/感情)が偏っていないか
- [ ] 伏線は「気づかれない程度に早く、忘れられない程度に近く」置けているか。回収は自然か
- [ ] クライマックスの解決が、主人公の選択・能力・これまでの積み重ねによるものか(偶然や第三者頼みになっていないか)
- [ ] テーマが説教ではなく状況と選択で表現されているか

## キャラクターレベル

- [ ] 主要キャラのWant/Needが対立構造として機能しているか
- [ ] セリフから名前を消しても、誰が言ったか分かるか(キャラクターボイスの差別化)
- [ ] プロットの都合でキャラの行動原理(禁則)を曲げていないか
- [ ] 敵役・障害となる人物に固有の動機があるか(単なる強さ比較になっていないか)
- [ ] 主人公としか繋がっていない孤立キャラはいないか

## 媒体レベル(マンガ/WEBTOON)

- [ ] セリフ・モノローグで説明しすぎていないか。セリフだけで話が全部分かるなら説明過多
- [ ] 1ページ・1スクロールの情報量が過密になっていないか(1ページ3〜6コマ、1フキダシ25〜30字目安)
- [ ] 掴み(第1話冒頭)が状況説明から入っていないか。絵になる状況から始まっているか
- [ ] 山場の決め絵がめくり/スクロールの快感位置に置かれているか
- [ ] 無言のコマ(間)がゼロになっていないか。感情の山場ほど絵に語らせているか
- [ ] キャラの位置関係がシーン内で矛盾なく追えるか

## 媒体レベル(小説・地の文)

- [ ] 視点(POV)がシーン内で滑っていないか。地の文が視点者の知らない情報を語っていないか
- [ ] 感情のラベリング(「悲しかった」等)に頼りすぎていないか。重要な感情ほど行動描写になっているか
- [ ] 一文の長さ・段落のリズムに単調さがないか。決定的な一文は際立っているか
- [ ] セリフがサブテキストを持たず、情報伝達だけになっていないか
- [ ] 五感が視覚に偏っていないか

## 文章の最終チェック

- [ ] 同じ語尾・接続詞が不自然に連続していないか
- [ ] 読点過多で一文が長くなりすぎていないか
- [ ] 「〜こと」「〜という」「〜のだ」等の癖が積もっていないか
- [ ] 声に出して読んでリズムが悪い箇所はないか
- [ ] 誤字脱字、固有名詞の表記ゆれ(キャラ名・地名・SE/M/心などの記法)がないか

## フィードバックを返すときの態度

編集者としてフィードバックする場合の詳細な作法は `pitch-planning.md` の「編集フィードバックの書き方」を参照。要点:

- 良かった点を「残すべき箇所の特定」として具体的に先に伝える
- 最重要の直しは1〜3点に絞る。全部に赤を入れない
- 「なぜ機能していないか(読者にどう見えるか)」+「改善の方向性」をセットで返す
- 書き手の意図が読めない箇所は、直しを出す前に意図を確認する
