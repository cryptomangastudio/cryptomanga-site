# 推敲・編集チェックリスト

原稿(プロット・台本・小説本文いずれも)を書き終えたら、最低1周はこの観点で見直す。全項目を毎回律儀にやる必要はないが、依頼の重要度に応じて厚みを調整する。

## マクロ(構造レベル)

- [ ] ログラインを1〜2文で言い直せるか。物語の芯がぶれていないか
- [ ] 主人公は受け身になっていないか(事件に反応するだけでなく、能動的に選択しているか)
- [ ] 「転」または中盤の転換で、価値観・関係性が実際に変化しているか
- [ ] 各話・各章の終わりに引きがあるか(連載/話数分割の場合)
- [ ] 伏線は「気づかれない程度に早く、忘れられない程度に近く」置けているか。回収は自然か
- [ ] テーマ(この物語で本当に言いたいこと)が、説教ではなく状況と選択で表現されているか

## キャラクターレベル

- [ ] 主要キャラのWant/Needが対立構造として機能しているか
- [ ] セリフから名前を消しても、誰が言ったか分かるか(キャラクターボイスの差別化)
- [ ] 敵役・障害となる人物に固有の動機があるか(単なる強さ比較になっていないか)
- [ ] 全員が同じ立場・同じ話し方になっていないか

## 媒体レベル(マンガ/WEBTOON)

- [ ] セリフ・モノローグで説明しすぎていないか。絵で語れる部分を文字で潰していないか
- [ ] 1ページ・1スクロールの情報量が過密になっていないか
- [ ] 掴み(第1話冒頭)が状況説明から入っていないか。絵になる状況から始まっているか
- [ ] コマ・スクロールの緩急(大ゴマ/密なコマ、長い間/速いテンポ)に意図があるか

## 媒体レベル(小説・地の文)

- [ ] 視点(POV)がシーン内で滑っていないか
- [ ] 感情のラベリング(「悲しかった」等)に頼りすぎていないか。行動描写で表現できる箇所はないか
- [ ] 一文の長さ・段落のリズムに単調さがないか
- [ ] セリフがサブテキストを持たず、情報伝達だけになっていないか

## 文章の最終チェック

- [ ] 同じ語尾・接続詞が不自然に連続していないか
- [ ] 読点過多で一文が長くなりすぎていないか
- [ ] 声に出して読んでリズムが悪い箇所はないか
- [ ] 誤字脱字、固有名詞の表記ゆれ(キャラ名・地名など)がないか

## フィードバックを返すときの態度

編集者としてフィードバックする場合、「良くない」だけで終わらせず、**なぜ機能していないか(上記チェックリストのどの観点に引っかかるか)**と、**改善の具体的な方向性**をセットで返す。書き手の意図を先に汲み取ってから直す(意図が読み取れない場合は先に確認する)。
