# 制作パイプライン — 1冊を「企画→出版→販促」まで流す標準工程

このパイプラインの目的はただ一つ:**あなた(創業者)の手を最小化しながら、Amazonに消されない・読者に返品されない品質の本を、反復可能な工程で量産する**こと。

原作(マンガ)と違い、Kindleの実用書・エッセイ・小説は「あなたの創作の魂」を毎回注ぐ必要はありません。その代わり、**工程を仕組みにして、判断ポイントだけあなたが握る**設計にします。

> ⚠️ 大前提: 「AIに丸投げして100冊」は **2024年以降ほぼ確実にAmazonに淘汰される**。ここでの"手を減らす"は「creativeな消耗を減らす」であって「品質チェックを飛ばす」ではありません。詳細は `docs/research/` と `docs/03_AI開示と規約遵守.md` を参照。

---

## 全体像(7ステップ)

```
① 企画      → どのジャンル・誰の悩み・何を約束する本か決める        [あなたの判断: 30分]
② 骨子      → 目次と各章の要点をAIと詰める                          [半自動]
③ 執筆      → 骨子に沿って本文をAI生成 + あなたが赤入れ             [自動+チェック]
④ 編集      → 事実確認・重複削除・トーン統一・独自性の注入          [ここが生命線]
⑤ 表紙+ページ → 表紙ブリーフ→表紙、販売ページのコピー              [半自動]
⑥ 出版      → KDPに入稿・AI開示・価格/KU設定・チェックリスト        [手動15分]
⑦ 販促+計測 → note連携・レビュー導線・収益トラッカー更新           [継続]
```

各ステップに対応するテンプレートが `templates/` にあります。

---

## ① 企画(あなたが判断する唯一の必須工程)

- `templates/書籍企画書.md` を1冊につき1枚うめる。
- 決めること: **ジャンル / 想定読者の1つの悩み / 読了後の変化(約束) / 競合3冊 / 差別化の一言 / 価格帯 / KU入れるか**。
- ⚠️ **書き始める前に「媒体」と「目標文字数」を必ず決める**。note用(数千字)かKindle用(2〜2.5万字)か両方か。そして**章ごとの字数配分表**を先に引く(例: `企画/第1巻_制作計画`)。これを飛ばすと"書いたら分量が全然足りない/多い"の事故になる(第1巻で実際に発生)。
- 判断基準は `docs/02_ジャンル・企画戦略.md`。「AIで薄い本を量産して沈むジャンル」を避け、**検索需要があり・返品されにくく・シリーズ化できる**テーマを選ぶ。
- 🚦 **Go/No-Go**: 競合上位が「AI量産本だらけで低評価」なら参入OK(改善余地)。逆に「熱心な個人が丁寧に書いた良書だらけ」なら勝ち目が薄い→別テーマへ。

## ② 骨子

- `templates/原稿骨子テンプレート.md` を使い、目次(章立て)+各章の「読者が得る1つのこと」を書き出す。
- プロンプトは `workflow/プロンプト集.md` の「骨子生成」。
- 分量の目安: 実用書は 15,000〜30,000字(KUのKENP読了で稼ぐなら厚め、単品売りなら中庸)。**水増しはKUでもバレる**(後半飛ばし読み=ページ既読が伸びない)。

## ③ 執筆

- 章ごとにAIで初稿を生成。**一気に全部生成しない**(章ごとに骨子とズレを確認)。
- 各章の初稿に必ず「あなた/担当者しか書けない一次情報」を1つ以上差し込む口を作る(体験・具体例・数字・失敗談)。これが④の独自性の素。

## ④ 編集(=生命線。ここを飛ばすと全部沈む)

Amazonの品質基準・返品・低評価は、ほぼこの工程の手抜きから来ます。最低限:

1. **事実確認**: AIが書いた固有名詞・数値・法律・手順を1つずつ検証(ハルシネーション除去)。実用書の誤情報はレビュー爆死+返品の直行便。
2. **重複・冗長の削除**: AI特有の「言い換え反復」を削る。読了率(KENP)に直結。
3. **トーン統一**: シリーズなら"語り手"を固定。`templates/` の文体メモを使う。
4. **独自性の注入**: ③で作った口に一次情報を入れる。**"検索1位の記事3本を混ぜた本"は独自性なしと判断されやすい**。
5. **リーガル**: 引用ルール・商標・画像権利・医療/投資/法律の断定回避。

## ⑤ 表紙 + 販売ページ

- 表紙: `templates/表紙ブリーフ.md` に「ジャンルの棚に馴染む+1点だけ目立つ」方針を書き、生成/外注。**表紙はクリック率を決める最重要資産**。ケチらない。
- 販売ページ: `templates/販売ページ.md`。冒頭3行(スマホで見える範囲)で「誰の何を解決するか」を言い切る。

## ⑥ 出版(KDP入稿)

- `docs/出版チェックリスト.md`(= このファイル下部)を上から順に。
- **AI開示**: KDPの入稿フォームでAI利用を申告(`docs/03_AI開示と規約遵守.md`)。`templates/AI利用記録.md` を1冊ごとに残す(監査・規約対応・確定申告の証跡)。
- 価格・KU: `docs/01_収益シミュレーション.md` の方針に従う。
- 出版後72時間はインデックス・カテゴリ反映を確認。

## ⑦ 販促 + 計測

- `tracker/書籍マスター.csv` に1行追加、`tracker/収益ダッシュボード.md` を月次更新。
- note連携: 本の1章を無料公開→本編へ誘導、or 制作の裏側を有料note化(`docs/04_note連携戦略.md`)。
- 30日・90日でKENP/売上を見て、**伸びないテーマは深追いせず次へ**、**伸びたテーマはシリーズ化**(量産の勝ち筋はシリーズ)。

## 出版前チェックリスト(⑥で使用)

- [ ] タイトル/サブタイトルに主要検索語が入っている(不自然な詰め込みはNG)
- [ ] 表紙がスマホのサムネ(小)でも判読できる
- [ ] 本文の誤情報チェック済み(固有名詞・数値・手順)
- [ ] 重複・冗長を削除、読了ストレスなし
- [ ] 独自性(一次情報)が最低1章に入っている
- [ ] 目次(TOC)が電子書籍として機能する(リンク)
- [ ] 冒頭に「本書はAIを活用して制作」等の透明性の一文(方針は03参照)
- [ ] KDP入稿フォームでAI利用を正しく申告
- [ ] 著作権・商標・画像権利をクリア(表紙素材のライセンス含む)
- [ ] 医療/投資/法律/健康は断定回避+免責文
- [ ] 価格・KU登録・カテゴリ・キーワード(7枠)設定
- [ ] `templates/AI利用記録.md` を保存
- [ ] `tracker/書籍マスター.csv` に登録

## 「手を減らす」ための現実的な分業

| 工程 | あなた | AI | 外注(任意) |
|---|---|---|---|
| ① 企画 | ◎ 判断 | 補助(競合調査) | — |
| ② 骨子 | チェック | ◎ | — |
| ③ 執筆 | 一次情報の口 | ◎ | — |
| ④ 編集 | ◎ 監修 | 補助 | 校正を外注可 |
| ⑤ 表紙 | 方針 | 生成 | デザイナー(推奨) |
| ⑥ 出版 | 実行 | — | VA(バーチャルアシスタント)可 |
| ⑦ 販促 | 方針 | 文面生成 | — |

**あなたが絶対に手放してはいけないのは ①企画 と ④編集の監修**。ここだけ握れば、残りは仕組みとAI/外注で回せます。原作より確実に軽い、というあなたの直感はこの2点に集約すれば正しい。
