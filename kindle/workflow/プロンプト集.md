# プロンプト集 — 制作パイプライン各工程で再利用するプロンプト

各プロンプトは「{ }」を自分の企画に置き換えて使う。1冊ごとに使い回すことで、品質のブレを抑える。
**出力は必ず④編集(事実確認・独自性注入)を通す前提。AI出力をそのまま入稿しない。**

---

## ① 企画: 競合リサーチ

```
Amazon Kindleストアの「{ジャンル/キーワード}」で、上位20冊を想定して分析して。
- 各本が読者に約束していること(ベネフィット)
- 価格帯とページ数の傾向
- レビューで褒められている点/不満点
- まだ誰も埋めていない「読者の悩み」の空白
最後に、この空白を突く新刊の切り口を3案、それぞれ一言で。
```

## ② 骨子生成

```
以下の企画で、Kindle実用書の目次(章立て)を作って。
- テーマ: {読者の1つの悩み}
- 想定読者: {誰}
- 読了後の変化(約束): {ビフォー→アフター}
- 分量: 本文 約{20,000}字 / {7〜10}章

条件:
- 各章に「読者がその章で得る具体的な1つのこと」を1文で添える
- 章の順序は「読者が実際に行動する順」に並べる
- 水増し・重複する章を作らない。薄いなら章を減らす
- 最初の章で「なぜ今これを読むべきか」を掴む
```

## ③ 章の初稿

```
次の章を、Kindle実用書の本文として書いて。
- 章タイトル: {章タイトル}
- この章で読者が得る1つのこと: {要点}
- 文体: {である調/ですます調}、{語り手の設定}
- 想定読者の前提知識: {初心者/中級}

守ること:
- 抽象論で終わらせず、具体例・手順・数字で示す
- 【一次情報】というマーカーを1〜2箇所置き、そこに私が実体験・独自の例を後で差し込めるようにする
- 断定できない事実(数値・法律・最新情報)は【要確認】と明記する
- 他章と重複しそうな内容は書かない
```

## ④ 編集: 事実確認の洗い出し

```
以下の本文から、事実確認が必要な記述をすべて抜き出してリスト化して。
- 固有名詞、数値、日付、統計、法律・制度、手順の正誤
- それぞれ「なぜ確認が必要か」「誤っていた場合のリスク」を添える
本文:
{本文}
```

## ④ 編集: 冗長・重複の圧縮

```
以下の本文を、意味を保ったまま冗長・言い換え反復・AI的な水増しを削って引き締めて。
- 削った箇所と理由を簡単にレポートして
- 読者が飛ばし読みしたくなる部分を指摘して
本文:
{本文}
```

## ⑤ タイトル/サブタイトル案

```
次の本のタイトルとサブタイトルを10案。
- 内容: {要約}
- 読者: {誰}
条件: 主要検索語を自然に含む / スマホのサムネで読める短さ / 誇大・断定(「必ず」「絶対」)を避ける / シリーズ化を想定した"型"も1〜2案入れる
```

## ⑤ 販売ページ(商品説明)

```
Kindleの商品説明文を書いて。
- 本の内容: {要約}
- 読者の悩み: {悩み}
- 読了後の変化: {約束}
構成:
1. 冒頭3行でスマホ画面に収まる「誰の何を解決するか」
2. この本で分かること(箇条書き5〜7）
3. 読者の変化(ビフォー→アフター)
4. こんな人に/こんな人には向かない(正直に)
5. 軽い行動喚起
誇大表現・断定・体験談の捏造はしない。
```

## ⑤ 表紙ブリーフ→画像プロンプト

```
Kindle表紙の画像生成プロンプトを作って。
- ジャンル: {ジャンル}（棚に馴染む色調・構図）
- 伝えたい感情: {安心/わくわく/権威/etc}
- タイトル文字を置く余白の位置: {上/中央}
- 避けたいこと: 文字化けする埋め込み文字、既存作品の模倣
英語プロンプトと日本語の意図メモの両方を出して。
```

## ⑦ note連携: 導線記事

```
Kindle新刊「{タイトル}」への導線になるnote記事の構成を作って。
- 本の1章分に相当する価値を無料で出し、続きは本編へ
- 押し売りにならず「役に立った→もっと読みたい」で終える
- 最後に本のリンクを自然に置く一文
```

---

## 使い方の原則

- **章単位で回す**。本1冊を一括生成しない(ズレ・重複・破綻が増える)。
- **【一次情報】【要確認】マーカー**を必ず残し、④で潰す。これが独自性と信頼性の源。
- プロンプトを改善したら、この集に反映して次の本から品質を底上げする(仕組みの複利)。
