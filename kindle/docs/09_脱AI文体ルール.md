# 09 脱AI文体ルール —「AIっぽさ」を消して、読まれる文章にする

> 全原稿(Kindle・note)に適用する必須ルール。CryptoManga Studiosの制作にある「**脱AI文体の番人**」を、この事業でも正式な査読視点として運用する。
> 発端: 第1巻まえがき初稿への代表の批評(2026-07)。「稼ぎ/損失の数字・家族の教訓・お節介アドバイスをミキサーにかけた、無難で綺麗なだけの副業本まえがき」。**書いた本人の顔と体温が見えない**ので読む気が起きない、という指摘。全部その通り。

---

## AIっぽさを生む5つの病(=禁止事項)

### 1. テンプレの「光→影→共感」構成
- ✗ 「実績で引く→挫折を語る→共感で締める」の黄金パターン。「稼いだ(光)/溶かした(影)」を綺麗な落差で並べる。
- ◎ **対比を綺麗に作らない**。語り順をわざとズラす(損失から始める/時系列を崩す/オチを先に言う)。教科書構成を意図的に壊す。

### 2. 記号的なエピソード(体温ゼロ)
- ✗ 大事件を"数字"として処理する。「700万溶かした」で終わり。感情語(絶望した/つらかった)だけで済ます。
- ◎ **その時、体と生活がどうだったか**を必ず入れる。胃、手、眠れたか、その日の家、誰に言えたか。**抽象の感情語でなく、身体・行動・具体物**で。ここは本人の実記憶が必須(AIが捏造しない)。

### 3. シグナルフレーズ(「これから本音ですよ」の合図)
- ✗ 「同じ口で、正直に言います」「ここだけの話」「正直に言うと」「実は」。本音に入る前の"合図"。
- ◎ **合図せず、いきなり言う**。本当のことは、前振りなしで置く。

### 4. 読者のコントロール(お節介・先回り)
- ✗ 「ちょっとだけ待ってください」「どうぞ肩の力を抜いて」「〜と思いませんか?」「気づきませんか」。読者の感情・態度を指定/代弁/同意誘導する。
- ◎ **読者の感情を指定しない**。事実と自分の話を置いて、感じ方は読者に委ねる。呼び止め・気遣いの押し付けをしない。

### 5. 整いすぎ(ノイズがない)
- ✗ 全文が均等な心地よいリズム。文頭「——」でタメ(多用)。「A。B、C、Dも。」の3〜4拍子の綺麗な羅列。引っかかりがなさすぎる。
- ◎ **ノイズを残す**。一文の長さを不均一に。ぶっきらぼうな断片、言い切り、主語の省略、小さな脱線。整いすぎたら、わざと崩す。「——」の連発禁止(1章に1回あるかないか)。3拍子の羅列を避ける。

---

## ペルソナ(語り手)を一つに固定する
- ✗ 「溶かしました」(くだけ)の直後に「肩の力を抜いて」(スマートなビジネス書作家)。口調が混ざると"不気味の谷"になる。
- ◎ **著者の実際の喋り方に寄せる**。この著者は、カジュアル・直球・自虐まじり(音声取材の口調)。ビジネス書のすまし顔にしない。全編、同じ人が喋っている状態を保つ。

## 避けたい"AI定番語"(乱発しない)
「〜という選択」「〜のポテンシャル」「本質」「〜と地続き」「〜に他ならない」「まさに」「ある種の」「〜と言えるでしょう」。滑らかで無難な言い回しは、AI臭の温床。

## 編集時の「脱AI文体の番人」パス(全原稿で実行)
`docs/07`§4の検証プロトコルに、この査読視点を1枠として加える。原稿を読み返し、次を潰す:

- [ ] 光→影→共感のテンプレになっていないか(なっていたら順序を崩す)
- [ ] 大事件に"体温"(身体・行動・具体物)があるか。なければ本人の実記憶を入れる(捏造しない)
- [ ] シグナルフレーズ(「正直に言うと」等)がないか
- [ ] 読者コントロール(「待って」「肩の力を抜いて」「〜ですよね?」)がないか
- [ ] 「——」の多用・3拍子の羅列・整いすぎがないか(ノイズを足す)
- [ ] 語り手の口調が最初から最後まで一人か
- [ ] AI定番語を乱発していないか

> 最重要: **本人の実記憶(体温)なしに、AIだけで"生々しさ"は作れない**。生々しさが要る箇所は、必ず取材(`取材質問リスト`)で本人の本当のことを引き出してから書く。想像で埋めない。
