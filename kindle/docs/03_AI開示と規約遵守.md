# 03 AI開示と規約遵守 — 「消されない」ための最重要ルール

> このフォルダの前提は「AI利用をちゃんと申告する」。それは正しい。**規約違反での垢BAN=全収益ゼロ**が、この事業で最も現実的で致命的なリスク。ここは絶対に手を抜かない。
> 各条項の最新の正確な文言は必ず一次ソース(KDPヘルプ)で確認すること。要点と運用は `docs/research/` の研究レポートに確定版を反映。

---

## KDPのAIコンテンツの考え方(2つの区別)

Amazon KDPは「AIコンテンツ」を2種類に分けている。**この区別が申告要否を決める。**

| 区分 | 定義(目安) | 開示 |
|---|---|---|
| **AI生成(AI-generated)** | テキスト・画像・翻訳をAIが作成したもの(あなたがその後に編集を加えても、元をAIが作ったなら"生成") | **申告が必要** |
| **AI支援(AI-assisted)** | あなたが自分で作成し、AITは着想・編集・推敲の補助に使ったもの | 申告不要 |

- 申告はKDPの**出版(タイトル作成)フォーム内**で行う。テキスト/画像/翻訳ごとに問われる。
- **開示内容は読者(購入者)には表示されない**。Amazon内部の管理用。だから「申告すると売れなくなる」心配は不要 → 正直に申告するのがノーリスク。
- 迷ったら**開示する側に倒す**。過少申告は規約違反リスク、過剰申告は実害なし。
- 1冊ごとに `templates/AI利用記録.md` を残し、自分の本が「生成/支援」どちらかを判断・記録する。

## 量産を規制する実務ルール(踏むと即アウト)

- **1日の新規タイトル登録数に上限がある**(スパム的大量出版の抑制策)。100冊は「1日で登録」ではなく**日々コツコツ積む**前提で計画する。
- **低品質・低コンテンツ(low-content/low-quality)本の取り締まり**: 中身が薄い/検索記事の焼き直し/内容と無関係なタイトル・キーワード等はブロック・削除対象。AIかどうか以前に"品質"で弾かれる。
- **重複・酷似コンテンツの禁止**: 既存本の焼き直し、他者著作の流用、AIで微差の量産はNG。シリーズは"型は同じでも中身は別物"であること。
- **誤解を招くメタデータの禁止**: タイトル/表紙/説明文と中身の不一致(釣り)は違反+返品+低評価の三重苦。
- **アカウント停止は連鎖する**: 1冊の違反で全アカウント停止・既刊も道連れになりうる。1発退場を避ける運用が最優先。

## 日本の著作権(AI生成物の扱い・要点)

- 日本の著作権法では、著作物は「思想又は感情を**創作的に表現**したもの」。**人間の創作的寄与**があってはじめて著作権が生じるのが基本的な考え方(文化庁の整理)。
- **AIが自律生成しただけの出力は、人間の創作的寄与が乏しいと著作権が認められない可能性**がある。=あなたの本を第三者にパクられても保護が弱くなりうる。
- 逆に言えば、**骨子設計・大幅な編集・独自情報の追加など人間の創作的関与を厚くするほど、著作物性が認められやすく、品質もAmazon対策も同時に満たす**。→ `workflow/制作パイプライン.md`④編集の重要性はここにも効く。
- **生成物が既存著作物に依拠・類似**していると侵害になりうる(AI利用でも通常の著作権侵害の判断は同じ)。生成物が既存作品に酷似していないか確認する。
- ※法解釈は発展途上。金額が大きくなる/紛争リスクの高いジャンルを扱う前に専門家に確認。

## 「手を減らす」と「規約を守る」を両立させる設計

矛盾しない。むしろ同じ方向:

1. **①企画と④編集(人間の創作的寄与)だけは握る** → 品質基準クリア・著作物性確保・差別化を同時達成。
2. **正直に開示する** → 開示は読者に見えず売上に響かない。過少申告のBANリスクだけ消える。
3. **1日数冊ペースで積む** → 登録上限に触れず、品質チェックの時間も確保。
4. **釣らない・焼き直さない・パクらない** → 一発退場の主要因を全部避ける。

> この4つを守れば、「AIで手を減らしつつ、消されずに積み上げる」は両立する。守らなければ、何冊書いても土台ごと消える。
