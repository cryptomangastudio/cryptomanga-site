# LINE Harness（ラインハーネス）徹底調査メモ（2026-07-19）

> 著者の問い：「無料のLINE公式アカウントのようなサービス"LINE Harness"に乗り換えた方がいいのでは？」
> 結論の先出し：**"乗り換え"ではない。** LINE HarnessはLINE公式アカウントの"代わり"ではなく、**LINE公式アカウントの上に乗せて機能拡張する無料ツール**（Lステップ・エルメと同じ立ち位置）。だから正しくは「LINE公式アカウント＋LINE Harness」。LINE公式アカウントは引き続き必須。

---

## 1. LINE Harnessとは（事実）
- **オープンソース（MIT）の無料LINE運用ツール**。2026年3月公開。開発＝Shudesu（野田修一）氏。GitHub 536★・v0.17.0（2026-07時点）＝活発に更新中。
- 提供形態＝**自分のCloudflare上に自分でデプロイして使う（セルフホスト）**。SaaSではない。
- 必要な前提：**①LINE公式アカウント ②Messaging APIチャネル ③Cloudflareアカウント ④Node.js 22/pnpm**。→ ①②が要る＝**LINE公式の代替ではなく拡張**。
- セットアップ：`npx create-line-harness` で約5分（Cloudflare認証・D1データベース作成・デプロイ・LINE認証情報登録を自動化）。技術スタック＝Cloudflare Workers＋Next.js＋SQLite(D1)。
- **Claude Code用のMCPサーバー同梱**＝自然言語でシナリオ作成・配信・会話一覧などを操作できる（＝今のClaude/私から運用可能）。

### 機能（Lステップ有料級を無料で）
配信＝ステップ配信・一斉/セグメント/タグ配信・予約・リマインダー・テンプレ・トラッキングリンク／CRM＝友だち管理・タグ・スコアリング・オペレーターチャット・重複検知／集客＝リッチメニュー・LIFFフォーム・カレンダー予約・スタッフ権限／自動化＝IF-THENルール・キーワード応答・Webhook／複数アカウント＝プール管理・**BAN検知→自動で次アカウントへ友だち移行**。

---

## 2. お金の本当のところ（最重要）
「無料」は**ツール利用料が0円**という意味。**LINEに払うメッセージ配信料は、どのツールを使っても同じだけかかる**（＝Harnessでは下がらない）。理由：配信は結局LINE公式アカウントのMessaging API枠を消費するから。

| 費目 | 金額 | 備考 |
|---|---|---|
| LINE Harness 利用料 | **¥0**（永久） | OSS |
| Cloudflare | **¥0**（友だち〜5,000人）／超過は1万人ごと約$10/月 | Cloudflareへの支払い（Harnessではない） |
| **LINEのメッセージ配信料** | **別途・全ツール共通** | 無料枠＝月200通まで。超過は従量課金（自動移行）。有料プラン=月¥5,000/¥15,000等。2026年10月改定で追加メッセージは20万通まで1通¥3・超過¥2.5に |
| 友だち数そのもの | ¥0 | 課金は"配信数（通数×人数）"に対して |

→ **要点：Harnessが節約するのは「ツールの月額（Lステップ¥5,000〜/エルメ¥10,780）」だけ。LINEの配信料は不変。** 小さいリスト（配信が月200通以内）なら、そもそも素のLINE公式が無料で足りる。

## 3. メリット／デメリット（この事業＝軌道キャリ視点）

### メリット
- **ツール月額が0円**でステップ配信・自動化・CRM・リッチメニュー・フォームまで使える（素のLINE公式には無い機能）。
- **Claude Code MCP同梱**＝私（Claude）から自然言語で運用・保守できる。"最小関与で回す"という本事業の方針と、実は相性が良い（＝人手をClaudeで代替できる）。
- 複数アカウント＋BAN検知自動移行＝配信でBANされやすいグレー運用への保険（本事業は健全運用なので優先度は低いが）。

### デメリット・リスク
- **セルフホスト＝セキュリティ責任が全部自分**。CloudflareアカウントとLINEアクセストークン、そして**友だち（顧客）の個人データ**を自分で守る必要。Cloudflareを乗っ取られたら顧客データに触られる。有料SaaSの月額には「セキュリティ・サポート」という見えない価値が含まれる、という指摘は妥当。
- **導入に技術知識**（Cloudflare・GitHub・CLI）。`npx`で自動化されるとはいえ、非エンジニア単独はややハードル高。→ **ただしClaude Code/私が代行できる**。
- **非常に新しい（2026年3月・v0.17）**＝実績・寿命・将来のメンテ継続に不確実性。顧客リストを預けるツールとしては、**エクスポートを定期的に取る**等の保険が要る。
- 更新・障害対応も自己責任（SaaSのような自動運用・問い合わせ窓口がない）。

## 4. 代替の選択肢（横並び）
| 選択肢 | ツール月額 | ホスティング/保守 | ステップ配信 | 向き |
|---|---|---|---|---|
| **素のLINE公式のみ** | ¥0 | LINE管理（不要） | ✗（無い） | 立ち上げ初期・小規模。今作った運用パックで即開始可 |
| **エルメ（L Message）無料** | ¥0（月1,000通まで） | SaaS（保守不要・安全） | ○ | 「managed＆安全＆無料でステップ配信」を最小手間で |
| **Lステップ / エルメ有料** | ¥5,000〜/¥10,780〜 | SaaS | ◎ | 規模が出て本格運用。月額を払う |
| **LINE Harness** | **¥0** | **自分のCloudflare（自己責任）** | ◎ | 月額0円で最大機能。ただし保守/セキュリティは自分（orClaude）持ち |

## 5. 推奨（結論）
1. **今（ローンチ〜小規模）＝素のLINE公式アカウントの無料枠でスタート**。作成済みの`LINE運用パック(全巻).md`がそのまま使える。設定ゼロ・リスクゼロ・費用ゼロ。友だちが少ないうちは、ステップ配信の自動化より「まず読者を集める」が先。
2. **友だちが増えて"ステップ配信の自動化"が欲しくなったら**、有料のLステップ/エルメを契約する前に、**LINE Harnessを最有力候補**に。理由＝①ツール月額0円 ②**MCPで私が構築・運用を代行できる**＝あなたの手離れが良い。導入は私が`npx create-line-harness`で進められる（要：あなたのCloudflareログインとLINE Messaging APIの認証情報。機密なので一緒に）。
3. **"managed＆安全"を最優先するなら**、間にエルメ無料プランを挟む手もある（保守不要・月1,000通まで）。セキュリティ責任を負いたくない場合はこちら。
4. どの道でも、**LINE公式アカウント自体は必要**で、**LINEのメッセージ配信料は共通**。だから「乗り換えで劇的にコストが下がる」わけではない（下がるのは"ツール月額"だけ）。

### 決め手の軸
- **最小関与でやりたい×Claudeに任せたい → LINE Harness（私が運用）** が本命。
- **自分もClaudeも保守で悩みたくない → エルメ無料 or 素のLINE公式**。
- いずれにせよ**顧客リストのエクスポートを定期取得**して、ツール消滅・障害の保険を持つ。
