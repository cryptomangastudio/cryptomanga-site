# Kindle × note 出版事業(土台)

CryptoManga Studios の「**創業者の手が最小で回る**」収益事業の土台。原作(マンガ)のように毎回あなたの創作の魂を注がなくても、**①企画の判断**と**④編集の監修**だけ握れば、残りは仕組み・AI・外注で回せる——という前提で設計している。

> このフォルダは「稼げます」と煽らない。**徹底リサーチ(59エージェント・306ツール実行)の検証結果に基づき、できること・できないことを正直に**置く。夢を見て100冊書いてゼロ、を避けるための土台。

---

## 先に結論(徹底リサーチの回答)

- **月10万円 … 条件つきで達成可能。ただし難易度は高い。**
- **年600万円(月50万円)… 個人が正攻法・持続的に到達するのは非現実的**(例外的成功者の水準)。
- **「AIで薄い本を100冊量産して稼ぐ」は成立しない。** 検証済みの3つの事実が否定する:
  1. **KUの単価が薄すぎる**(既読1ページ≒**0.478円**。150p完読で約72円)
  2. **1日3点までしか新規出版できない**(反AI大量出版キャップ、2023年9月〜継続)
  3. **手軽なジャンルほど凍結する**(AI写真集等がわいせつ規約違反で垢BAN多発 → 既刊も全滅)
- **勝ち筋** = 「**AI"補助"(開示義務の外)× 専門性・付加価値 × KDPとnoteの分散**」。冊数ではなく、**"当たる型を1つ作って複製"**。

詳細と根拠 → `docs/research/2026-07_Kindle・note収益化 徹底リサーチ.md`
数字での検証 → `docs/01_収益シミュレーション.md`

## あなたの直感は、半分正しい

「Kindleは原作よりテコ入れが少なくて済む」——**創作の消耗という意味では正しい**。毎回ゼロから世界観を生む必要はなく、工程を仕組み化できる。
ただし「**手を抜く**」という意味なら**間違い**。AI丸投げの薄い本はAmazonに淘汰され、読者に返品され、規約違反で消える。**"減らせるのは創作の消耗、減らせないのは品質チェック(④編集)"**。ここを取り違えなければ、あなたの見立ては筋が良い。

## 始め方(最初の30日)

1. **読む**: `docs/research/` → `docs/01` → `docs/02`。まず「何が成立して何が成立しないか」を頭に入れる(ここを飛ばすと100冊ゼロコース)。
2. **選ぶ**: `docs/02` を見ながら、検索需要があり・返品されにくく・シリーズ化できるジャンルを **3〜5テーマ**選ぶ。
3. **企画する**: 各テーマで `templates/書籍企画書.md` を埋め、Go判定が出たものだけ着手。
4. **作る**: `workflow/制作パイプライン.md` に沿って1冊通す。**①企画と④編集だけあなたが握り**、②③⑤はAI+テンプレで回す。
5. **出す**: `docs/出版チェックリスト.md` を上から。AI利用は正直に開示(`docs/03`)。
6. **測る**: `tracker/` に記録。30日で「動いた本」を見つけ、**動いた型だけを複製・シリーズ化**。動かない本は深追いしない。

> **合言葉**: 「先に"当たる1冊"と"その型"。冊数は最後。」当たりのない100冊は、100個のゼロ。

## 手を減らすための分業(あなたが握るのは2点だけ)

| 工程 | あなた | AI | 外注(任意) |
|---|---|---|---|
| ① 企画 | ◎ 判断 | 競合調査 | — |
| ②③ 骨子・執筆 | 一次情報の口だけ | ◎ | — |
| ④ 編集 | ◎ 監修 | 補助 | 校正 |
| ⑤ 表紙・ページ | 方針 | 生成 | デザイナー(推奨) |
| ⑥⑦ 出版・販促 | 実行/方針 | 文面 | VA |

**握るのは ①企画 と ④編集の監修 だけ。** ここだけで品質・独自性・著作物性・規約遵守が同時に立つ。残りは仕組みに委ねてよい——それが「原作より軽い」の正体。
